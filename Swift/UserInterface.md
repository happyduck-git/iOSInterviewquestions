18. 뷰 컨트롤러 간의 데이터 전달 및 통신을 어떻게 관리하셨나요?
    - 데이터를 전달하는 데 사용한 패턴이나 기법은 무엇이었나요?
VC간 결합도를 낮게 유지하면서도 데이터 전달 및 통신이 이루어질 수 있도록 하기 위해 주로 delegate패턴을 사용하거나 Combine과 같은 framework를 주로 활용하였습니다.
\
Delegate pattern을 사용하려면, protocol을 정의해주고 그 안에 delegator가 수행할 작업들의 function signature를 작성해줍니다. 실제적인 function에 대한 구현은 delegator가 구현하면 됩니다. delegate pattern이 다른 방식들 보다 코드를 더 많이 작성한다고도 하지만, compiler가 필수 function 구현 여부를 체크해주기 때문에 놓치지 않고 구현할 수 있다는 편리함으로 자주 사용하였습니다.
\
\
Combine과 같은 리액티브 framework를 사용해서도 객체 간 데이터 전달이 가능합니다. 데이터를 보내는 쪽을 publisher, 받는 쪽을 subscriber라고 표현을 한다면, 그 둘의 관계를 이용해서 데이터를 전달하고 구독하게 됩니다. Combine은 property wrapper를 사용해서 간략하게 데이터 전달을 할 수 있다는 장점이 있어서 사용하고 있습니다.
